# 02 WBS 文档

> 本 WBS 围绕“校园活动报名与签到管理系统”的项目交付物进行分解，共三级。

## 1. 一级交付物

1. 项目管理成果
2. 系统需求与设计成果
3. 活动发布与管理功能成果
4. 学生报名与审核功能成果
5. 签到与统计功能成果
6. 测试与验收成果

## 2. 二级与三级分解

### 1. 项目管理成果
- **1.1 启动文档**
  - 1.1.1 项目章程（01-project-charter.md）
  - 1.1.2 团队角色与分工表
- **1.2 计划文档**
  - 1.2.1 WBS 文档（02-wbs.md）
  - 1.2.2 简化进度计划（03-schedule.md）
- **1.3 配置与总结文档**
  - 1.3.1 配置管理计划（04-config-plan.md）
  - 1.3.2 实验总结报告（05-summary-report.md）
- **1.4 仓库与协作环境**
  - 1.4.1 Git 仓库与目录结构
  - 1.4.2 分支策略与提交记录

- **2.1 需求说明**
  - 2.1.1 功能范围列表（纳入/不纳入）
  - 2.1.2 用户角色描述（管理员/学生/签到人员/维护人员）

  - 2.2.1 活动报名业务流程说明
  - 2.2.2 签到业务流程说明

### 3. 活动发布与管理功能成果

- **3.2 活动列表与详情**
  - 3.2.1 活动列表展示规则
  - 3.2.2 活动详情查看规则

### 4. 学生报名与审核功能成果
- **4.1 学生报名**
  - 4.1.1 报名信息字段（学号、姓名、联系方式等）

### 5. 签到与统计功能成果

  - 5.1.2 扫码签到流程说明
- **5.2 签到记录管理**
  - 5.2.1 签到状态（已签到/未签到/迟到等）

### 6. 测试与验收成果

- **6.2 验收标准**
  - 6.2.1 各角色验收清单

## 3. 分解说明
本 WBS 严格遵循“按交付物分解”原则，一级条目对应项目的六大成果域，二级细化到具体交付物，三级则细化到可管理的文档或产出。所有条目均为名词性成果，避免使用“开会”“讨论”等动作描述，确保每一项都有明确的产出物或文档与之对应。
