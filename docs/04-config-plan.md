# 04 配置管理文档

## 1. 配置项范围
- README
- 项目启动说明
- WBS 文档
- 进度计划文档
- 配置管理文档
- 总结报告


## 2. 分支策略
- main: 受保护分支，仅允许 Merge Request 合并。
- dev: 日常开发集成分支。
- feature-*: 功能开发分支。

## 3. 命名约定
### 文档命名：
 严格按照实验要求，使用 两位数字序号-功能描述.后缀 格式。
例如：01-readme.md, 02-wbs.md, 03-schedule.md, 04-config-plan.md, 05-summary-report.md。
### 分支命名： 
依据《实验指导书》第5点“制定配置管理方案”。
main (主分支，受保护)
dev (开发主分支)
feature-[功能名] (功能分支，例如 feature-wbs, feature-schedule)
### 提交说明：
 依据《实验指导书》第2点“编写项目概况与协作约定”。
示例：[Add] Initial commit, [Update] Fix typo in WBS, [Merge] Resolve conflict
## 4. 合并规则
日常开发： 所有成员在各自的 feature-* 分支开发，完成后合并到 dev 分支。
最终交付： 实验结束时，将 dev 分支合并到 main 分支。
强制要求： 在合并前，必须确保完成了冲突解决（实验要求必须制造并解决一次冲突）。
## 5. 版本留痕
- **过程留痕：** 每次关键文档（WBS、计划）更新需提交一次。
- **最终版本：** 所有成果最终需合并至 `main` 分支。
- **实验要求：** 必须保留一次“冲突解决”的完整提交记录，用于实验报告说明。
