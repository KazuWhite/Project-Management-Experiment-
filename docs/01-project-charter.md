- # 01 项目启动说明

  ## 1. 项目名称
  校园活动报名与签到管理系统

  

  ---

  ## 2. 项目背景

  学校希望建设一套轻量级校园活动管理平台，解决当前活动组织过程中信息分散、报名混乱、签到效率低下等痛点。系统面向校内活动组织场景，强调流程清楚、记录完整和使用便捷。本实验不要求完成完整业务系统的开发，而是围绕该项目情境完成项目管理文档的编写与 Git 协作流程的实践。

  

  ---

  ## 3. 项目目标

  ### 3.1 业务目标

  | 编号 | 目标描述                                                     | 优先级 | 验收标准                       |
  | ---- | ------------------------------------------------------------ | ------ | ------------------------------ |
  | G1   | 支持活动管理员发布活动信息，维护报名时间、人数上限与活动地点 | 高     | 文档中有活动信息字段定义       |
  | G2   | 支持学生查看活动列表并完成在线报名                           | 高     | 文档中有报名流程描述           |
  | G3   | 支持活动组织者对报名名单进行审核与查看                       | 中     | 文档中有审核流程说明           |
  | G4   | 支持现场签到，并形成签到结果统计                             | 高     | 文档中有签到方式与统计字段定义 |
  | G5   | 支持活动结束后导出基本统计结果                               | 中     | 文档中有导出字段说明           |

  ### 3.2 管理目标

  - 所有项目文档纳入 Git 版本管理，保留完整修改历史
  - 采用分支协作流程，模拟真实团队开发场景
  - 每位成员均有清晰的提交记录，体现个人贡献
  - 最终交付物结构清晰，在 main 分支统一汇总

  

  ---

  ## 4. 主要使用角色

  | 角色         | 职责描述                                   |
  | ------------ | ------------------------------------------ |
  | 活动管理员   | 发布活动、维护活动信息、查看报名与签到统计 |
  | 学生         | 查看活动、提交报名、查询个人报名结果       |
  | 现场签到人员 | 核验学生身份并完成签到记录                 |
  | 系统维护人员 | 维护系统参数与基础配置                     |

  ## 6. 项目边界

  ### 6.1 纳入范围

  - 活动信息发布与维护
  - 学生报名信息提交
  - 报名名单查看与审核
  - 签到记录与签到状态管理
  - 签到统计结果导出
  - 项目文档的编写与 Git 版本管理
  - 分支协作流程的完整实践

  ### 6.2 不纳入范围

  - 复杂支付功能
  - 与真实校园统一身份认证系统的深度对接
  - 高并发优化与正式部署方案
  - 移动端 App 正式开发
  - 系统实际代码的编写与测试
  - UI 界面原型或高保真设计
