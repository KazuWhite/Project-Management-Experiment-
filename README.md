# 校园活动报名与签到管理系统

> 软件项目管理课程综合实验 · 软件工程23级 · 三人协作项目

---

## 1.项目概述

本项目为软件项目管理课程综合实验。项目以"校园活动报名与签到管理系统"
为统一背景，不要求开发完整业务系统，而重点训练以下核心能力：

- **范围管理**：围绕交付物开展三级 WBS 分解
- **进度管理**：提取关键活动，建立简化进度计划
- **配置管理**：建立受控版本库，制定分支策略
- **协作能力**：多人分支协作、冲突制造与解决

## 5. 协作约定
提交规范：提交信息使用 docs: <具体修改内容> 格式，如 docs: 完成三级WBS分解；冲突解决提交信息以 conflict: 开头。

合并规则：个人分支合并到 dev 前需先推送至远程，由配置管理员在本地拉取并合并；dev 合并到 main 须经全组复查后由配置管理员执行。

身份要求：每位成员严格使用本人姓名拼音设置 user.name，提交前用 git log -1 --format='%an %ae' 自查。

冲突处理：合并时若发生冲突，禁止直接推送，需在本地拉取最新 dev 后手工解决，确认无误后再推送。

---
## 6. 分支管理策略

本项目采用 Git Flow 简化模型：

feature → dev → main

规则如下：
- 所有开发必须基于 feature 分支进行
- 完成后合并至 dev 分支进行集成
- dev 分支稳定后合并至 main 作为最终交付
- main 分支禁止直接提交

冲突处理：
- 冲突由配置管理员统一协调解决
- 所有冲突需记录在 conflict.md 中

## 8. 提交历史说明

所有成员均在 feature-* 分支上完成各自的文档初稿，并合并到 dev 进行集成。冲突文件 conflict.md 由两位成员分别修改同一行，在合并到 dev 时触发冲突，随后通过VS Code手动解决并提交。最终 dev 分支的内容被完整合并到 main，形成本仓库的最终提交版本。完整的提交记录和协作痕迹可在 commits 页面和 Git 图表中查看。
