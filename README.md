# 校园活动报名与签到管理系统

> 软件项目管理课程综合实验 · 软件工程23级 · 三人协作项目

---

## 1.项目概述

本项目为软件项目管理课程综合实验。项目以"校园活动报名与签到管理系统"
为统一背景，不要求开发完整业务系统，而重点训练以下核心能力：

- **范围管理**：围绕交付物开展三级 WBS 分解
- **进度管理**：提取关键活动，建立简化进度计划
- **配置管理**：建立受控版本库，制定分支策略
- **协作能力**：多人分支协作、冲突制造与解决

## 4. 分支约定
| 分支名 | 用途 | 说明 |
|------|------|------|
| `main` | 最终交付分支 | 只接受来自 dev 的合并 |
| `dev` | 集成分支 | 各 feature 分支完成后合并至此 |
| `feature-charter` | 启动说明 | 赖瑞林负责 |
| `feature-plan` | WBS文档 | 罗成瑞鑫负责 |
| `feature-readme` | README | 赖瑞林负责 |
| `feature-plan` | 进度计划 | 罗成瑞鑫负责 |
| `feature-config` | 配置管理 | 李卓卿负责 |
| `feature-conflict-*` | 冲突练习 | 罗成瑞鑫与李卓卿各一个 |

---
## 5. 分支管理策略

本项目采用 Git Flow 简化模型：

feature → dev → main

规则如下：
- 所有开发必须基于 feature 分支进行
- 完成后合并至 dev 分支进行集成
- dev 分支稳定后合并至 main 作为最终交付
- main 分支禁止直接提交

冲突处理：
- 冲突由配置管理员统一协调解决
- 所有冲突需记录在 conflict.md 中
